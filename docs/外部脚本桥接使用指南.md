# 外部脚本桥接使用指南

## 概述

OpsCopilot 支持集成外部定位脚本来增强问题排查能力。通过桥接脚本模式，你可以将任何外部工具集成到 OpsCopilot 中。

## 工作原理

```
OpsCopilot → 桥接脚本 (接收固定参数) → 外部定位工具 (输出结论)
```

1. OpsCopilot 调用桥接脚本，传递两个固定参数：
   - `-Problem`: 问题描述
   - `-OutputDir`: 输出目录路径

2. 桥接脚本调用你的外部定位工具

3. 外部工具在输出目录中创建 `conclusion.md` 文件

4. OpsCopilot 读取结论并与 AI 分析结果综合

## 快速开始

### 1. 配置桥接脚本路径

在 OpsCopilot 设置中，配置"外部定位脚本路径"，指向你的桥接脚本：
- 例如：`C:\tools\opscopilot_bridge.bat`

### 2. 创建桥接脚本

复制 `scripts/troubleshoot_bridge_template.bat` 并根据你的需求修改：

```batch
@echo off
REM 解析参数
set "PROBLEM=%~1"
set "OUTPUT_DIR=%~2"

REM 调用你的外部工具
python "C:\tools\my_analyzer.py" "%PROBLEM%" "%OUTPUT_DIR%"

REM 或者直接写逻辑
echo # 结论 > "%OUTPUT_DIR%\conclusion.md"
echo 分析结果... >> "%OUTPUT_DIR%\conclusion.md"
```

### 3. 输出格式要求

外部工具需要在 `%OUTPUT_DIR%` 目录下创建 `conclusion.md` 文件，格式建议：

```markdown
# 定位结论

## 问题分析
分析内容...

## 排查步骤
1. 步骤一
2. 步骤二

## 建议命令
\`\`\`bash
command1
command2
\`\`\`
```

## 示例场景

### 场景1: 集成 Python 分析脚本

```batch
@echo off
set "PROBLEM=%~1"
set "OUTPUT_DIR=%~2"

python "C:\tools\analyze.py" --problem "%PROBLEM%" --output "%OUTPUT_DIR%\conclusion.md"
```

### 场景2: 调用企业内部 API

```batch
@echo off
set "PROBLEM=%~1"
set "OUTPUT_DIR=%~2"

curl -X POST "https://your-api/troubleshoot" \
  -d "problem=%PROBLEM%" \
  -o "%OUTPUT_DIR%\response.json"

REM 转换 JSON 为 Markdown
python convert_json_to_md.py "%OUTPUT_DIR%\response.json" > "%OUTPUT_DIR%\conclusion.md"
```

### 场景3: 直接调用日志分析工具

```batch
@echo off
set "PROBLEM=%~1"
set "OUTPUT_DIR=%~2"

REM 根据问题关键词分析日志
findstr /C:"%PROBLEM%" "C:\logs\app.log" > "%OUTPUT_DIR%\log_snippet.txt"

REM 生成结论
(
echo # 日志分析结论
echo.
echo ## 问题
echo %PROBLEM%
echo.
echo ## 相关日志
echo \`\`\`
type "%OUTPUT_DIR%\log_snippet.txt"
echo \`\`\`
) > "%OUTPUT_DIR%\conclusion.md"
```

## 测试

使用测试脚本验证配置：

```bash
# 手动测试桥接脚本
scripts\test_troubleshoot.bat
```

## 参数说明

### 接收参数

| 参数 | 说明 | 示例 |
|------|------|------|
| `-Problem` | 问题描述，用户输入的问题文本 | "服务器 CPU 占用率过高" |
| `-OutputDir` | 输出目录路径，用于存放结论文档 | `C:\Users\用户\AppData\Local\Temp\OpsCopilot\troubleshoot` |

### 输出文件

- **文件名**: `conclusion.md`
- **位置**: `%OutputDir%\conclusion.md`
- **格式**: Markdown 格式

## 常见问题

### Q: 桥接脚本执行失败怎么办？

A: 检查以下几点：
1. 确认脚本路径在设置中配置正确
2. 确认脚本有执行权限
3. 手动运行脚本验证参数解析是否正确

### Q: 如何调试桥接脚本？

A: 在脚本中添加调试输出：
```batch
echo Debug: Problem=%PROBLEM% >> "%OUTPUT_DIR%\debug.log"
echo Debug: OutputDir=%OUTPUT_DIR% >> "%OUTPUT_DIR%\debug.log"
```

### Q: 支持调用 PowerShell 脚本吗？

A: 支持。在桥接脚本中调用：
```batch
powershell -ExecutionPolicy Bypass -File "C:\tools\my_script.ps1" -Problem "%PROBLEM%" -OutputDir "%OUTPUT_DIR%"
```
