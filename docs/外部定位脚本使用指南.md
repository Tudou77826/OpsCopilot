# 外部定位脚本使用指南

## 概述

OpsCopilot 支持外部定位脚本功能，可以在 AI 分析的同时并行运行自定义脚本，提供额外的定位思路。

## 快速开始

### 1. 配置脚本路径

1. 打开 OpsCopilot
2. 点击设置（齿轮图标）
3. 在"应用选项"标签页中找到"外部定位脚本路径"
4. 输入脚本完整路径，例如：
   - `.bat` 文件：`D:\scripts\troubleshoot.bat`
   - `.ps1` 文件：`D:\scripts\troubleshoot.ps1`

### 2. 启用增强模式（可选）

如果需要向脚本传递额外参数：

1. 打开故障排查面板
2. 在"开始排查"按钮上方勾选 **"外挂脚本增强模式"**
3. 配置参数：
   - **代码仓库路径**（可选）：例如 `D:\dev\my-project`
   - **自定义变量**（可选）：点击"编辑"添加键值对

### 3. 开始排查

1. 在问题输入框中输入问题描述
2. 点击"开始排查"
3. OpsCopilot 和外部脚本将**并行执行**
4. 查看结果：
   - **OpsCopilot** 标签页：AI 分析结果
   - **外部定位** 标签页：脚本输出结果
   - **综合答复** 标签页：AI 整合后的结论

## 参数说明

程序会自动向脚本传递以下参数：

### 基本参数（始终传递）

| 参数 | 说明 | 示例 |
|------|------|------|
| `-Problem` | 用户输入的问题描述 | `-Problem "CPU占用率过高"` |
| `-OutputDir` | 临时输出目录 | `-OutputDir "C:\Users\xxx\AppData\Local\Temp\OpsCopilot\troubleshoot"` |

### 增强模式参数（启用后传递）

| 参数 | 说明 | 示例 |
|------|------|------|
| `-RepoPath` | 代码仓库路径 | `-RepoPath "D:\dev\my-project"` |
| `-自定义变量名` | 用户添加的自定义变量 | `-Branch "main" -Env "prod"` |

## 脚本开发指南

### Batch 脚本示例 (.bat)

```batch
@echo off
setlocal enabledelayedexpansion
chcp 65001 >nul 2>&1

REM Initialize variables
set "PROBLEM="
set "REPO_PATH="
set "OUTPUT_DIR="

REM Parse arguments
:parse_args
if "%~1"=="" goto :end_parse
if "%~1"=="-Problem" (
    set "PROBLEM=%~2"
    shift
    shift
    goto :parse_args
)
if "%~1"=="-RepoPath" (
    set "REPO_PATH=%~2"
    shift
    shift
    goto :parse_args
)
if "%~1"=="-OutputDir" (
    set "OUTPUT_DIR=%~2"
    shift
    shift
    goto :parse_args
)
shift
goto :parse_args

:end_parse

REM Output result in Markdown format
echo # External Tool Conclusion
echo.
echo ## Analysis
echo Problem: !PROBLEM!
echo.
echo ## Findings
echo - Issue 1
echo - Issue 2
```

### PowerShell 脚本示例 (.ps1)

```powershell
param(
    [string]$Problem,
    [string]$RepoPath,
    [string]$OutputDir
)

# Output result in Markdown format
Write-Output "# External Tool Conclusion`n"
Write-Output "## Analysis`n"
Write-Output "Problem: $Problem`n"
if ($RepoPath) {
    Write-Output "Repository: $RepoPath`n"
}
Write-Output "## Findings`n"
Write-Output "- Issue 1`n"
Write-Output "- Issue 2`n"
```

### 输出格式要求

脚本应该输出 **Markdown 格式**的文本，这样可以：
- 在"外部定位"标签页中正确显示
- 被 AI 正确解析并整合到"综合答复"中

建议包含以下部分：
```markdown
# 分析标题

## 问题分析
详细的问题分析...

## 排查思路
- 检查项 1
- 检查项 2

## 建议命令
```bash
command1
command2
```
```

## 手动测试脚本

在开发或调试脚本时，可以手动运行测试：

```cmd
REM 测试 .bat 脚本
test_troubleshoot.bat -Problem "Test Issue" -RepoPath "D:\test" -OutputDir "C:\Temp"

REM 测试 .ps1 脚本
powershell -NoProfile -ExecutionPolicy Bypass -File test.ps1 -Problem "Test Issue" -RepoPath "D:\test" -OutputDir "C:\Temp"
```

## 常见问题

### Q: 脚本执行失败，提示 "exit status 0xfffd0000"
**A:** 检查脚本文件扩展名：
- `.bat` 文件使用 `cmd.exe` 执行
- `.ps1` 文件使用 `powershell.exe` 执行
- 确保脚本路径正确且有执行权限

### Q: 脚本无法接收参数
**A:** 检查参数解析逻辑：
- Batch 脚本使用 `%~1`, `%~2` 等位置参数
- PowerShell 脚本使用 `param()` 块定义参数
- 确保参数名称正确（区分大小写）

### Q: 中文输出乱码
**A:** 在脚本开头添加编码设置：
- Batch: `chcp 65001 >nul`
- PowerShell: 使用 UTF-8 编码保存文件（BOM）

### Q: 如何查看脚本执行错误？
**A:** 查看"外部定位"标签页，如果有错误会显示错误信息和脚本路径。

## 最佳实践

1. **输出格式统一**：使用 Markdown 格式，便于 AI 解析
2. **错误处理**：脚本应该处理异常情况并输出友好的错误信息
3. **参数验证**：验证必需的参数是否存在
4. **输出到文件**（可选）：利用 `-OutputDir` 参数保存中间结果
5. **测试充分**：手动测试脚本确保参数解析正确
