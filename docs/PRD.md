# 产品需求分析书 (PRD)

**项目名称**：OpsCopilot (智能运维终端)
**版本号**：v1.0
**日期**：2026-01-03

## 1. 项目背景与目标
针对现有终端软件（如 Xshell, iTerm2）在复杂运维场景下的局限性，开发一款集成 **AI 智能辅助** 与 **批量自动化控制** 的下一代终端工具。旨在降低运维门槛，解决记忆复杂命令困难、多节点操作繁琐、故障定位依赖人工经验等痛点。

## 2. 核心用户画像
*   **运维工程师 (SRE)**：需要频繁操作多台服务器，进行日志排查和批量变更。
*   **后端开发人员**：需要快速定位线上问题，依赖业务文档和特定排查指令。

## 3. 功能需求详情

### 3.1 模块一：AI 智能业务助手 (RAG & NLP)

*   **F1.1 自然语言转连接配置 (Smart Connect)**
    *   **描述**：用户输入自然语言（如：“连一下生产环境支付节点 10.0.1.5，跳板机是 office-bastion，密码 admin/123”），AI 自动提取目标 IP、跳板机信息、账号密码。
    *   **输出**：弹出一个已预填好的连接配置确认框。
*   **F1.2 业务故障定位 (RAG)**
    *   **描述**：结合本地导入的“业务排查文档”，用户询问故障现象（如“订单创建超时”），AI 检索文档并生成排查思路。
    *   **特性**：支持本地向量库，数据不出域（可选）。
*   **F1.3 动态命令生成**
    *   **描述**：AI 提供的命令不只是文本，而是**参数化模板**（如 `grep {{order_id}} error.log`），用户在 UI 填参后一键执行。

### 3.2 模块二：高级连接管理 (Connectivity)

*   **F2.1 复杂链路一键连接**
    *   **描述**：支持“本地 -> SSH 跳板机 -> SSH 目标节点 -> `su root` 提权”的全自动连接流程。
    *   **要求**：支持配置多层跳板机，支持自动识别 `Password:` 提示符并输入 Root 密码。
*   **F2.2 场景化工作区 (Workspace)**
    *   **描述**：支持保存“NCE集群001”场景，包含 4 个节点的连接配置。点击场景一键并发启动 4 个会话。

### 3.3 模块三：多路复用与广播 (Multiplexing)

*   **F3.1 多屏监控布局**
    *   **描述**：支持 2x2 网格分屏，或“主 Tab + 状态概览”模式，实时查看 4 个节点的输出流。
*   **F3.2 命令广播 (Broadcasting)**
    *   **描述**：支持通过开关控制：开启后输入命令，实时发送到当前工作区的所有激活终端，关闭时则只发送到当前活动终端。
