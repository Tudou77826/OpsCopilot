### 1. 前端组件修改：设置模态框 (`SettingsModal.tsx`)
*   **更新 Props 接口**：增加 `isBroadcastMode` (boolean) 和 `onToggleBroadcast` (function) 属性，允许从外部控制广播状态。
*   **新增标签页**：在现有的 `llm`, `prompts`, `system` 之外，增加 `app` (应用选项) 标签页。

### 2. 前端核心逻辑修改：主应用 (`App.tsx`)
*   **状态管理**：
    *   新增 `isBroadcastMode` 状态，默认为 `false`（不持久化，重启重置）。
    *   新增 `broadcastIds` 状态，类型为字符串数组，存储当前参与广播的终端 ID。
*   **广播控制逻辑**：
    *   实现 `handleToggleBroadcast(enabled)`：
        *   开启时：设置 `isBroadcastMode = true`，并将当前**所有**活跃终端的 ID 填入 `broadcastIds`。
        *   关闭时：设置 `isBroadcastMode = false`，并清空 `broadcastIds`。
    *   实现 `handleToggleTerminalBroadcast(id)`：允许在广播模式下，单独切换某个终端是否接收广播。
*   **输入处理逻辑 (`handleTerminalData`)**：
    *   拦截终端输入。
    *   判断：如果 `isBroadcastMode` 为真 **且** 当前终端 ID 在 `broadcastIds` 列表中。
    *   执行：调用后端 `App.Broadcast(broadcastIds, data)`，将输入分发给组内所有终端。
    *   否则：维持原有逻辑，调用 `App.Write(id, data)` 仅发送给自己。
*   **清理工作**：
    *   在 `handleCloseTerminal` 中，当终端关闭时，将其 ID 从 `broadcastIds` 中移除。
    *   删除 `BroadcastBar` 组件及其引用。

### 3. 前端界面修改：布局管理器 (`LayoutManager.tsx`)
*   **接收新 Props**：`isBroadcastMode`, `broadcastIds`, `onToggleTerminalBroadcast`。
*   **渲染逻辑**：
    *   **仅当 `isBroadcastMode === true` 时**：
        *   在 Tab 模式的标签页标题旁显示广播图标（📡）。
        *   在 Grid 模式的终端标题栏显示广播图标。
        *   图标状态：
            *   **激活（绿色/高亮）**：ID 存在于 `broadcastIds` 中。
            *   **未激活（灰色）**：ID 不在 `broadcastIds` 中。
        *   交互：点击图标调用 `onToggleTerminalBroadcast`。
    *   **当 `isBroadcastMode === false` 时**：不渲染任何额外图标，保持界面整洁。
