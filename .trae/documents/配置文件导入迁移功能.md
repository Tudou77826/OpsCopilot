## 实现配置文件导入迁移功能

### 功能概述
在设置页面的"应用设置"中添加配置导入功能，允许用户指定一个旧版本目录，从中读取配置文件并完成软件升级迁移。

---

## 实现步骤

### 1. 后端 Go 代码 - pkg/config/store.go
- 添加 `ImportFromDirectory(dirPath string) error` 方法
- 实现配置合并策略：
  - LLM、Prompts、QuickCommands、HighlightRules：旧配置覆盖
  - 其他配置：保留新版本默认值

### 2. 后端 Go 代码 - app.go
- 添加 `ImportConfigFromDirectory(dirPath string) string` 导出方法

### 3. 前端 TypeScript - SettingsModal.tsx
- 在"应用设置"标签页添加"导入旧版本配置"表单组
- 添加目录输入、选择按钮、导入按钮、状态显示

### 4. 前端 TypeScript 类型定义
- 添加 ImportConfigFromDirectory 方法类型声明

## 技术细节

**配置合并优先级**：
- 用户自定义配置（旧目录） > 当前默认配置
- 新版本新增字段保留默认值

**错误处理**：
- 目录不存在：提示用户检查路径
- 配置文件损坏：跳过该文件，继续导入其他文件
- JSON 解析失败：返回详细错误信息

**安全考虑**：
- 仅支持 JSON 格式文件
- 路径验证，防止目录遍历攻击
- 导入前可备份当前配置
