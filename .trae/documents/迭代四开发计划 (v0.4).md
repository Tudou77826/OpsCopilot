# 迭代四：AI 智能业务助手 - 核心篇 (v0.4) 开发计划 (Revised)

本计划旨在引入 LLM 能力，实现自然语言解析连接意图 (Smart Connect)，特别支持**批量连接**场景。

## 1. 后端开发：LLM 客户端封装
**目标**：提供统一的 LLM 调用接口，支持 OpenAI 格式的 API。
- [ ] **Config**: 添加 `pkg/config` 模块，用于读取 API Key 和 Base URL。
- [ ] **Define**: 定义 `LLMProvider` 接口 (`ChatCompletion`)。
- [ ] **Implement**: 实现 `OpenAIProvider`。

## 2. 后端开发：智能意图解析 (Smart Connect)
**目标**：将自然语言转换为**一组**结构化的连接配置 (`[]ConnectConfig`)。
- [ ] **Prompt Engineering**: 设计 System Prompt，要求输出 JSON **数组**。
    -   支持单条指令解析出多个目标主机。
    -   智能推断：如果未明确指定每个主机的账号密码，默认使用上下文中提到的通用凭据。
    -   结构示例：`[{"host": "172.16.0.5", ...}, {"host": "172.16.0.6", ...}]`。
- [ ] **Service**: 实现 `AIService.ParseConnectIntent(input string) ([]ConnectConfig, error)`。
- [ ] **Test**: 编写 `pkg/ai/intent_test.go`。
    -   输入："通过 10.0.0.1 连接 172.16.0.5 到 .8" -> 验证返回数组长度为 4，且均包含跳板机配置。

## 3. 前端开发：AI 对话组件 (SmartConnectModal)
**目标**：提供对话框，支持解析结果的**批量预览与确认**。
- [ ] **Component**: 创建 `SmartConnectModal`。
    -   **结果预览区**：不再是单表单，而是一个**连接列表**（Table 或 List）。
    -   **批量操作**：允许用户勾选/取消勾选解析出的特定主机。
    -   **手动修正**：点击列表项可编辑单个配置（如修正 IP 或密码）。
    -   **一键连接**：点击 "Connect All" 后，前端循环调用 `App.Connect` 批量发起连接。
- [ ] **Integration**: 在 App 顶栏添加 "Smart Connect" 按钮入口。

## 4. API 暴露与集成
- [ ] **API**: 在 `App.go` 中暴露 `ParseIntent(input string) []ConnectConfig` 方法。
- [ ] **Flow**: 前端调用 `ParseIntent` -> 后端请求 LLM -> 返回 Config 数组 -> 前端展示列表 -> 用户确认 -> 批量连接。

## 验收标准
- [ ] **单点连接**：输入 "连接 192.168.1.5"，解析出 1 个配置。
- [ ] **批量连接**：输入 "连接 192.168.1.10, 1.11, 1.12"，解析出 3 个配置。
- [ ] **复杂场景**：输入 "通过跳板机 10.0.0.1 (root/123) 连接内网 172.16.0.[5-8] (admin/abc)"，解析出 4 个配置，且均带有正确的跳板机和目标机认证信息。
