# 终端搜索与高亮：可执行测试用例

本文档覆盖以下功能的可执行测试：终端搜索（Ctrl+F）、搜索导航/计数、规则高亮、设置项开关、规则管理与配置落盘。

---

在 `d:\dev\workspace-go\OpsCopilot\frontend` 执行：

```bash
npm run build
npx vitest run
```

预期：
- build 成功
- vitest 全部通过

### 1.2 后端单测

在 `d:\dev\workspace-go\OpsCopilot` 执行：

```bash
go test ./...
```

预期：
- 全部通过

## 2. 手工测试（桌面应用真实验收）

### 2.1 前置条件

- 已启动应用
- 至少建立 1 个 SSH 连接并打开终端（确保能看到终端输出）
- 设置中“终端搜索”开启（默认开启）
- 设置中“终端高亮”开启（默认开启）

---

## 3. 搜索（Ctrl+F）测试

### TC-S-001 打开/关闭搜索面板--OK

步骤：
1. 在终端区域点击使其获得焦点
2. 按 Ctrl+F
3. 观察左下角弹出搜索面板
4. 点击终端区域使搜索框失去焦点
5. 按 Esc

预期：
- 第 2 步后，搜索面板出现且输入框获得焦点
- 第 5 步后，搜索面板关闭，终端仍可输入

### TC-S-002 输入查询与计数显示--OK

步骤：
1. Ctrl+F 打开面板
2. 输入 `ssh`（或任何会在终端输出中出现的词）
3. 等待 0.5 秒

预期：
- 计数栏出现数字或 `>=1000`（代表匹配数达到上限）
- 终端中所有命中位置出现高亮（亮黄色底色）
- 过程中 UI 无明显卡顿/“刷新感”

### TC-S-003 导航：输入框内 Enter/Shift+Enter + 按钮--OK

步骤：
1. Ctrl+F 打开面板并输入一个存在多次匹配的词
2. 在搜索输入框内按 Enter 多次【按下后终端换行了】
3. 在搜索输入框内按 Shift+Enter 多次【按下后终端换行了】
4. 点击“▶”多次【都是黄底的，根本看不出当前是哪一个】
5. 点击“◀”多次

预期：
- Enter 每次跳转到下一个匹配
- Shift+Enter 每次跳转到上一个匹配
- 点击“▶”每次跳转到下一个匹配
- 点击“◀”每次跳转到上一个匹配
- 所有命中为亮黄色底色；当前命中为橙色底色（用于区分当前位置）
- 终端输入不受影响（Enter 仍用于在终端执行命令）

### TC-S-005 防止“页面刷新/系统查找”回归--OK

步骤：
1. Ctrl+F 打开面板，使光标在搜索输入框内
2. 再次按 Ctrl+F
3. 输入框内按 Esc
4. 在终端内按 Enter（确保仍能正常执行）

预期：
- 不出现浏览器/系统的查找 UI
- 不出现“页面刷新”或明显闪烁
- Esc 仅用于关闭搜索面板，不干扰终端
- Enter 不会被搜索功能拦截

## 4. 规则高亮测试

### TC-H-001 新建规则并看到高亮

步骤：
1. 打开设置（⚙️）→ 应用选项 → 点击“管理高亮规则”
2. 点击“+ 添加规则”
3. 设置：
   - 名称：Error
   - 正则：`(?i)\\berror\\b`
   - 背景色：`#5a1d1d`
   - 文字色：`#ffffff`
4. 勾选启用
5. 回到终端，输出包含 `error` 的文本（例如运行会打印 error 的命令/日志）

预期：
- 终端中包含 `error` 的位置出现背景色/前景色高亮

### TC-H-002 全局关闭高亮

步骤：
1. 设置 → 应用选项 → “高亮已开启”开关关闭
2. 观察终端中已有高亮

预期：
- 已有高亮消失
- 后续输出不会再出现高亮

### TC-H-003 优先级（上下箭头）生效

步骤：
1. 添加两条规则：
   - RuleA：正则 `(?i)\\berror\\b` 背景红，优先级 10
   - RuleB：正则 `(?i)\\b(error|warn)\\b` 背景黄，优先级 0
2. 确保两条都启用
3. 让终端输出包含 `error`

预期：
- `error` 匹配处优先显示优先级更高的规则（数值更小者）

### TC-H-004 高风险正则强制禁用

步骤：
1. 管理高亮规则 → 新建规则
2. 正则输入：`(.+)+`
3. 尝试启用

预期：
- 出现风险提示
- 该规则无法被启用（或启用后自动回到禁用状态）

## 5. 配置落盘测试

### TC-P-001 保存规则后重启仍生效

步骤：
1. 创建并启用一条可见高亮规则
2. 点击“保存设置”
3. 关闭应用并重新启动
4. 打开终端输出相同内容

预期：
- 规则仍存在且仍启用
- 高亮效果仍存在

### TC-P-002 highlight_rules.json 文件生成

步骤：
1. 运行应用并保存任意高亮规则
2. 在应用工作目录查找 `highlight_rules.json`

预期：
- 文件存在
- 内容为 JSON 数组，包含刚保存的规则

---

## 6. 回归检查清单（每次改动必跑）

- 搜索面板：Ctrl+F 打开、Esc 关闭、Enter/F3 导航、不会触发系统查找/页面刷新
- 计数：输入查询不会明显卡顿；关闭面板计数任务停止
- 高亮：可见区内能出现高亮；滚动/新输出会更新；关闭高亮立即停止
- 设置：规则管理可保存；高风险正则不可启用
- 落盘：重启后仍生效
