# 实现问题排查时的并行外部定位工具集成（最终版）

## 需求分析

1. 在问题排查开始时，并行启动一个可配置的 Windows 脚本文件
2. 此功能只在问题排查场景触发，不在 AI 问答中触发
3. **每个视图独立管理加载状态**：哪个完成就可以立即切换查看，不需要等所有都完成
4. 问题排查答复中提供切换按钮，可切换查看：OpsCopilot、其他定位助手结论、综合答复

## 实现计划

### 1. 后端配置修改 (pkg/config/store.go)

* 在 `ExperimentalConfig` 结构体中添加 `ExternalTroubleshootScriptPath` 字段

* 在 `NewManager()` 中设置默认值为空字符串

* 更新配置导入逻辑，支持从旧版本导入该配置

### 2. 后端核心逻辑修改 (app.go & pkg/ai/)

* 创建新的结构体 `TroubleshootResult`：

  ```go
  type TroubleshootResult struct {
      OpsCopilotAnswer   string `json:"opsCopilotAnswer"`
      ExternalAnswer      string `json:"externalAnswer"`
      IntegratedAnswer    string `json:"integratedAnswer"`
      OpsCopilotReady   bool   `json:"opsCopilotReady"`
      ExternalReady      bool   `json:"externalReady"`
      IntegratedReady    bool   `json:"integratedReady"`
      ExternalError      string `json:"externalError,omitempty"`
  }
  ```

* 修改 `AskTroubleshoot` 方法签名，接收外部脚本路径

* 实现并行执行逻辑（无超时限制）：

  1. 使用 goroutine 并行执行两个任务：

     * **任务1**：调用 `aiService.AskTroubleshoot` 获取 OpsCopilot 结论

     * **任务2**：通过 `os/exec.Command` 执行外部脚本

       * 捕获 stdout 作为外部结论

       * 捕获 stderr 作为错误信息

  2. 使用 `select` 语句实时接收结果：

     * OpsCopilot 完成：设置 `OpsCopilotAnswer` 和 `OpsCopilotReady = true`

     * 外部脚本完成：设置 `ExternalAnswer` 和 `ExternalReady = true`

     * 外部脚本执行失败：设置 `ExternalError` 和 `ExternalReady = true`

  3. 生成综合答复（仅当两者都完成时）：

     * 使用 AI 将两个结果合并为 `IntegratedAnswer`

     * 设置 `IntegratedReady = true`

  4. 返回完整的 `TroubleshootResult` JSON

* **关键点**：

  * 不设置超时，允许外部脚本运行任意长时间

  * 使用 channel 实时传递结果

  * 一旦某个任务完成，立即将状态返回给前端

* 添加视图切换 UI（在 AI 回复区域上方）：

  ```typescript

    <button 
      onClick={() => setViewMode('opscopilot')}
      disabled={!troubleshootResult?.OpsCopilotReady}

    </button>
    <button 
      onClick={() => setViewMode('integrated')}
  ```

## 关键技术点

1. **真正的并行执行**：使用 Go goroutine + select 模式

2. **独立状态管理**：每个视图各自独立显示，无需等待全部完成

3. **实时反馈**：加载中状态显示 ...，完成显示 ✓

4. **错误容错**：外部工具失败不影响主流程

5. **用户体验**：哪个完成就可以立即切换查看

## 实现顺序

1. 后端配置和核心逻辑（优先）
2. 前端设置界面
3. 前端排查面板
4. 测试和调优
